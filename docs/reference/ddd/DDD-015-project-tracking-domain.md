## TL;DR for newcomers

*Skip if you already know the project-tracking bounded context.*

This DDD captures the Project Tracking and Status Federation bounded context: the part of the system that owns what agentbox knows about the projects it watches ‚Äî local git repositories and GitHub-mounted checkouts ‚Äî and how that knowledge is published natively on agentbox's three existing substrates (URN grammar, port-bound Prometheus telemetry, custom-kind nostr mesh). The pain point is that helm-grade project dashboards (status grid, 30-day commit activity, AI primers and synopses, GitHub plus local repo sync) are valuable, but helm ships them on a Fastify/React stack with no telemetry and no nostr. agentbox already has the better substrate. The shape of the answer is a domain that adopts helm's tracking model and re-expresses it sovereignly: every watched project becomes a first-class `TrackedProject` URN minted through `management-api/lib/uris.js`; every scan is a PROV-O `ProjectScan` activity receipt; the 30-day commit chart is a derived `CommitActivityWindow` read-model; AI primers and synopses persist only through the existing memory adapter slot; and a project's current status federates as an addressable kind-30841 `TrackingDigest` nostr event ‚Äî a sibling of the kind-30840 session summary.

**If you remember only one thing:** project tracking is read-mostly observability, not a new adapter slot. It mints no new URN kinds and opens no new durable backend ‚Äî it reuses the existing eighteen kinds and routes durable state through the memory and events slots it already has. The label that ever leaves the box is the project *slug*, never a host path; the only external hop is gated GitHub enrichment; and the owner identity is always the public `did:nostr:<AGENTBOX_PUBKEY>`.

For the deep version, keep reading.

## Domain Purpose

The truth this domain owns is the current, evidence-backed status of every project agentbox watches. "Watches" means: a directory containing a `.git` is discovered under a manifest-configured scan root, its git history is read (read-only ‚Äî this domain never writes to a git repository), and a `TrackedProject` projection is materialised against the operator's sovereign identity. The status is not an opinion; it is read out of `git log`, `git rev-parse`, and the optional GitHub API, then projected into a stable URN-addressed aggregate.

Three things make this a domain rather than a script. First, identity: a project is not a path ‚Äî it is a content-addressed URN that stays stable across rescans and across the local-versus-GitHub-mount distinction. Second, evidence: every scan emits a `ProjectScan` activity receipt, so "when was this last looked at, and what did the look produce" is itself a first-class, PROV-O-aligned record. Third, federation: a project's status is meant to be *seen* ‚Äî on the existing `/metrics` endpoint by a monitoring stack, and on the nostr mesh by the operator's mobile client ‚Äî without inventing a single new substrate.

Nothing in this domain mutates a watched repository, generates code, or persists outside the slots agentbox already owns. AI primers and synopses are an enrichment over the projection, generated by an inference call and stored in the memory adapter; they are not the source of truth. The source of truth is git.

## Bounded Context Definition

**Boundary**: This domain owns project discovery, project identity, scan receipts, the derived commit-activity read-model, memory-backed primers and synopses, the project-metrics projection, and the tracking-digest federation event.

**Owns** (IN):

- The `TrackedProject` aggregate ‚Äî its identity (a content-addressed `thing` URN), lifecycle, and projection fields.
- The `ProjectScan` activity ‚Äî the PROV-O action receipt for one scan pass over one or more scan roots.
- The `CommitActivityWindow` read-model ‚Äî the derived 30-day, per-day commit histogram for a project.
- `ProjectPrimer` and `ProjectSynopsis` ‚Äî memory-backed AI enrichments of a project, persisted only through the memory adapter slot.
- The `TrackingDigest` ‚Äî the human-readable, addressable kind-30841 status event for one project.
- The `ProjectMetricsProjection` ‚Äî the `agentbox_project_*` Prometheus series registered on the shared `/metrics` registry.
- `PrimerStatus` ‚Äî the lifecycle enum of a project's primer (`none` | `pending` | `active` | `error`).

**Does not own** (OUT):

- Vector storage primitives. RuVector is consumed as a port (the memory adapter slot, ADR-005) under a single namespace `project-tracking-primers`. This domain never issues raw SQL and never opens its own store.
- JSON-LD encoding. DDD-004 owns the encoding surface; this domain emits plain domain projections and exposes an opt-in JSON-LD port, encoded only when `[linked_data]` is on.
- Nostr cryptography, signing, relay transport, and pod-mailbox durability. DDD-003 and the Rust `services/nostr-pod-bridge` own these. This domain produces an unsigned `TrackingDigest` payload and hands it to the bridge's `track` subcommand; the nsec never crosses into this domain.
- Git itself. This domain is a strictly read-only consumer of git via `child_process.execFile` (no shell). It does not own repository state, branching, or commit creation.
- Privacy redaction policy. ADR-008 owns the filter; this domain routes adapter writes through it as usual and contributes the privacy constraint that the project label is always the slug (I02).
- The GitHub API contract. GitHub is an external system reached through the `gh` CLI under a gated enrichment port; its schema is not this domain's to define.

## Aggregates

### TrackedProject (Root)

The `TrackedProject` is the consistency boundary and the primary aggregate of this domain. It is agentbox's stable, content-addressed projection of one watched repository.

**Identity**: `urn:agentbox:thing:<scope>:project-<sha256-12>` ‚Äî minted through `management-api/lib/uris.js` against the `thing` kind. `<scope>` is the 64-character BIP-340 x-only hex pubkey (`AGENTBOX_PUBKEY`); the `project-<sha256-12>` local segment carries the kind discriminator prefix plus the content hash of the project's *remote URL* when one exists, falling back to the *absolute path* when the repository has no remote. Using the remote in preference to the path means a project keeps one identity whether it is scanned as a local checkout or as a GitHub mount (I01, I05).

**Fields**:

| Field | Type | Notes |
|---|---|---|
| `id` | `string` | Short stable handle (the `project-<sha256-12>` local segment) used in route paths. |
| `urn` | `urn:agentbox:thing:‚Ä¶` | Full canonical identity, minted via `uris.js`. |
| `ownerDid` | `did:nostr:<hex>` | `did:nostr:<AGENTBOX_PUBKEY>`. Public. (I08) |
| `name` | `string` | Display name; derived from the slug. |
| `path` | `string` | Absolute host path. Internal only ‚Äî never emitted as a metric label or nostr tag (I02). |
| `source` | `'local' \| 'github-mount'` | Discovery provenance. |
| `remote` | `string \| null` | Git remote URL if any; basis for content-addressed identity. |
| `branch` | `string` | Current branch from `git rev-parse`. |
| `language` | `string \| null` | Heuristic primary language. |
| `lastCommitIso` | ISO-8601 `\| null` | Timestamp of the most recent commit. |
| `lastCommitAgeSec` | `number \| null` | Seconds since the most recent commit, at scan time. |
| `commits30d` | `number` | Total commits in the trailing 30-day window. |
| `commitDays` | `[{date, count}]` | The `CommitActivityWindow` projection (see below). |
| `openIssues` | `number \| null` | From `GithubEnrichment`; `null` when not enriched. |
| `stars` | `number \| null` | From `GithubEnrichment`; `null` when not enriched. |
| `primerStatus` | `PrimerStatus` | `none` \| `pending` \| `active` \| `error`. |
| `primerUrn` | `urn:agentbox:memory:‚Ä¶` `\| null` | The active `ProjectPrimer` URN if one exists. |
| `scannedAt` | ISO-8601 | Timestamp of the scan that produced this projection. |

**Lifecycle states**:

```
Undiscovered ‚Üí Discovered ‚Üí Scanned ‚Üí Enriched ‚Üí Primed
                                ‚Üë          ‚îÇ         ‚îÇ
                                ‚îî‚îÄ‚îÄ(rescan)‚î¥‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
                                            ‚îÇ
                                       Untracked

Undiscovered : a .git directory exists under a ScanRoot, not yet projected
Discovered   : URN minted; basic git metadata read
Scanned      : full git projection (commits30d, CommitActivityWindow) materialised; metrics emitted
Enriched     : optional GithubEnrichment overlaid (openIssues, stars)
Primed       : a current ProjectPrimer/ProjectSynopsis pair exists in memory (primerStatus=active)
Untracked    : project no longer present under any ScanRoot; metrics cleared (clearProject)
```

Rescan is idempotent: re-scanning a project whose determining content (remote/path) is unchanged returns the same identity URN and overwrites the prior projection in place (I05). A project that disappears from all scan roots transitions to `Untracked` and its metric series are cleared.

**Invariants**:

- **I01**: Every `TrackedProject` URN is minted through `management-api/lib/uris.js` `mint()` against the `thing` kind with the owner pubkey as scope. Ad-hoc template-literal or `format!()` URN construction is prohibited. The local segment is content-addressed (ADR-013 R1) on the remote URL when present, else the absolute path.
- **I02**: The project label in every projection that leaves the box ‚Äî the `project` metric label, the kind-30841 `d`-tag, and the digest content name ‚Äî is the slug (the directory basename), never the absolute host path. The `path` field exists on the aggregate for internal correlation only and is excluded from telemetry and nostr surfaces by construction.
- **I05**: A scan is idempotent on the content-addressed identity. Two scans of the same project (same remote, or same absolute path when remote-less) produce the same `TrackedProject` URN and the second overwrites the first in place ‚Äî no duplicate aggregates accumulate.
- **I08**: `owner_did` on every `TrackedProject` (and every other record this domain emits) is `did:nostr:<AGENTBOX_PUBKEY>` ‚Äî the public x-only pubkey hex. It is a federation identity, not a secret; no nsec is ever read by this domain.

### ProjectScan

`ProjectScan` is an immutable activity record: the PROV-O action receipt for one scan pass. It records the act of looking, separately from the `TrackedProject` projections that the look produces. Nothing downstream is permitted to claim "a scan happened" other than via a `ProjectScan` record.

**Identity**: `urn:agentbox:activity:<scope>:projscan-<sha256-12>` ‚Äî minted via `uris.js` against the `activity` kind, content-addressed over the scan inputs (roots + start timestamp), with the `projscan-` discriminator prefix.

**Fields**:

| Field | Type | Notes |
|---|---|---|
| `scanUrn` | `urn:agentbox:activity:‚Ä¶` | Canonical identity. |
| `ownerDid` | `did:nostr:<hex>` | Public owner identity. |
| `roots` | `string[]` | The scan roots walked. |
| `discovered` | `number` | Count of projects projected by this scan. |
| `durationMs` | `number` | Wall-clock duration of the pass. |
| `outcome` | `'success' \| 'error'` | Whether the pass completed; per-repo failures do not fail the pass (I06). |
| `startedAt` | ISO-8601 | Timestamp at scan start. |

**Invariants**:

- **I06**: A scan is fail-open per repository. A repository that cannot be read (corrupt `.git`, permission error, git timeout) is logged, emits a `ProjectScanFailed`-scoped per-repo event, and is skipped ‚Äî it never aborts the surrounding scan pass. The pass `outcome` is `success` as long as the pass machinery completed; only an enclosing-pass failure yields `outcome: 'error'`.
- **I10**: Every scan pass emits exactly one `ProjectScan` activity record and exactly one `agentbox_project_scans_total{outcome}` counter increment. The scan duration is observed once into `agentbox_project_scan_duration_seconds`. A scan that produces no projects still emits its receipt.

### CommitActivityWindow (read-model)

The `CommitActivityWindow` is a derived read-model: the trailing 30-day, per-day commit histogram for one project, computed from `git log` at scan time. It is the sovereign re-expression of helm's 30-day commit chart.

**Identity**: `urn:agentbox:dataset:<scope>:commits-<projsha>-30d` ‚Äî minted via `uris.js` against the `dataset` kind, where `<projsha>` is the owning project's content hash. The `30d` suffix and `commits-` prefix are stable, so the window URN is deterministic from the project identity.

**Shape**: an ordered array `[{date: 'YYYY-MM-DD', count: number}]` of exactly 30 entries (days with no commits carry `count: 0`), most recent last. Exposed through `ProjectTracker.activity(id) ‚Üí {project, window: '30d', days: [...]}`.

**Invariants**:

- **I09**: `CommitActivityWindow` is a derived read-model and never a write source. It is rebuilt in full from `git log` on every scan; it is never mutated incrementally, never persisted as durable truth independent of git, and never the basis for any other aggregate's state. The window is always exactly the fixed trailing 30 days relative to scan time.

### ProjectPrimer / ProjectSynopsis (memory-backed)

`ProjectPrimer` and `ProjectSynopsis` are the AI enrichments of a project. The primer is a longer-form orientation document; the synopsis is its one-to-three-sentence distillation, carried in the kind-30841 digest content. They are generated by a single inference call against the project's git facts and persisted only through the memory adapter slot.

**Identity**: `urn:agentbox:memory:<scope>:primer-<sha256-12>` and `urn:agentbox:memory:<scope>:synopsis-<sha256-12>` ‚Äî minted via `uris.js` against the `memory` kind, content-addressed over the generated text.

**Generation**: performed by `PrimerGenerator.primer(project)` (`lib/project-primer.js`), which calls the configured `ZAI_URL /v1/messages` anthropic-shaped endpoint (`x-api-key` + `anthropic-version`), mirroring `config/hooks/nostr-session-summary.py`, under a two-slot concurrency limit (helm's `withAISlot` pattern). When the generator is not configured, it returns nulls and `PrimerStatus = none` ‚Äî it never blocks a scan.

**Invariants**:

- **I04**: Primers and synopses persist only through the memory adapter slot (ADR-005), under the single namespace `project-tracking-primers`, using the canonical memory MCP tools ‚Äî never raw SQL, never a bespoke store. The memory URN is minted via `uris.js`. A write that cannot route through the adapter leaves `primerStatus = error` and is not silently persisted elsewhere.
- **I07**: GitHub enrichment is the only external network hop in this domain and is gated. It runs only when `[project_tracking].github_enrichment = true` and a `GITHUB_TOKEN` is present, reached through the `gh` CLI under `execFile` (no shell). Primer generation (an inference call to the configured `ZAI_URL`) is the only other outbound call and is itself gated by generator configuration; both are fail-open and neither runs when its gate is off.

### TrackingDigest (federation event)

The `TrackingDigest` is the addressable kind-30841 nostr event projecting one project's current status to the mesh. It is the sibling of the kind-30840 session summary and follows the same dual-write-then-publish path through the Rust bridge.

**Identity**: `urn:agentbox:event:<scope>:projtrack-<sha256-12>` ‚Äî minted via `uris.js` against the `event` kind, content-addressed over the digest payload. This is the agentbox-internal URN; it is distinct from the nostr event id, which the bridge computes at signing time (DDD-003).

**Kind and addressing**: `KIND_PROJECT_TRACKING = 30841`, NIP-33 (parameterised replaceable). Tags: `["d", slug]` (addressable identity ‚Äî a re-publish replaces the prior digest), `["p", recipient_hex]`, `["t", "agentbox-project"]`, plus `["r", remote]` and `["l", language]` when present, and `["alt", "Project status: <name>"]`. Content is the human-readable digest: name, synopsis, language, last commit, `commits30d`, open issues, stars, primer status, and the project URN.

**Invariants**:

- **I03**: A `TrackingDigest` is addressable by `d`-tag = slug. Re-publishing a project's digest replaces the prior one for that slug under NIP-33; the mesh always holds at most one current digest per tracked project. The `d`-tag is the slug, never a host path (consistent with I02).
- **I11**: Nostr publication is a fail-open optional egress. It runs only when `[project_tracking].nostr_publish = true` and the bridge secrets are present; otherwise it is a silent no-op recorded as `agentbox_project_nostr_publish_total{outcome="skipped"}`. A publish success or failure is recorded as `success` or `error` respectively. A publish failure never fails a scan or a route call. Signing, dual-write to the pod inbox and `projects/<id>.jsonld`, and relay transport are all owned by `services/nostr-pod-bridge`'s `track` subcommand ‚Äî this domain only hands over the unsigned payload.
- **I12**: Kind 30841 is a reuse of the existing `event` URN kind and the existing nostr substrate ‚Äî no new agentbox URN kind and no new adapter slot is minted for project tracking. The kind number is registered in `[sovereign_mesh.relay].allowed_kinds` alongside 30840. This follows the Code-as-Harness URN-reuse precedent (DDD-005, PRD-008): new capabilities reuse the eighteen existing kinds rather than expanding the grammar.

## ProjectMetricsProjection

The `agentbox_project_*` series are a pure projection of `TrackedProject` and `ProjectScan` state onto Prometheus. They are registered on the **shared** registry imported from `management-api/observability/metrics`, so they appear on the existing port-bound `/metrics` endpoint (9090 in-process, 9091 standalone, bound `0.0.0.0`) with no new exporter. The projection is owned here; the registry and exporter are owned by ADR-005 observability.

| Series | Type | Labels | Source field |
|---|---|---|---|
| `agentbox_project_tracked_total` | Gauge | ‚Äî | count of `TrackedProject` |
| `agentbox_project_info` | Gauge (=1) | `project, language, source, owner_did, urn` | identity projection |
| `agentbox_project_commits_30d` | Gauge | `project` | `commits30d` |
| `agentbox_project_open_issues` | Gauge | `project` | `openIssues` (enriched) |
| `agentbox_project_stars` | Gauge | `project` | `stars` (enriched) |
| `agentbox_project_last_commit_age_seconds` | Gauge | `project` | `lastCommitAgeSec` |
| `agentbox_project_primer_status` | Gauge (1 for active status) | `project, status` | `primerStatus` |
| `agentbox_project_scan_duration_seconds` | Histogram | ‚Äî | `ProjectScan.durationMs` |
| `agentbox_project_scans_total` | Counter | `outcome` (`success`\|`error`) | `ProjectScan.outcome` |
| `agentbox_project_nostr_publish_total` | Counter | `outcome` (`success`\|`error`\|`skipped`) | `TrackingDigest` publish |

Privacy constraint (I02): the `project` label is always the slug; the only identity-bearing label is `owner_did`, which is the public pubkey. No absolute host path appears in any series. When a project transitions to `Untracked`, `clearProject(slug)` removes its gauge series so stale projects do not linger on the scrape.

## Domain Events (operator-visible)

| Event | Trigger | Key Payload Fields |
|---|---|---|
| `ProjectDiscovered` | A `.git` directory is found under a `ScanRoot` and a URN is minted | `project_urn`, `slug`, `source`, `owner_did` |
| `ProjectScanned` | Full git projection materialised for a project | `project_urn`, `slug`, `commits30d`, `last_commit_age_seconds`, `branch`, `language` |
| `ProjectScanFailed` | A repository cannot be read during a pass (per-repo, fail-open) | `path_hash`, `reason` ‚Äî repo skipped, pass continues (I06) |
| `ProjectScanCompleted` | A scan pass finishes | `scan_urn`, `roots`, `discovered`, `duration_ms`, `outcome` |
| `GithubEnrichmentApplied` | Gated GitHub enrichment overlays a project | `project_urn`, `open_issues`, `stars` |
| `GithubEnrichmentSkipped` | Enrichment gate off, no token, or non-GitHub remote | `project_urn`, `reason` ("disabled" \| "no-token" \| "no-github-remote") |
| `PrimerGenerated` | A primer/synopsis pair is written to memory | `primer_urn`, `synopsis_urn`, `project_urn`, `model` |
| `PrimerGenerationSkipped` | Generator not configured | `project_urn` ‚Äî `primerStatus` stays `none` |
| `PrimerGenerationFailed` | Inference or memory write fails | `project_urn`, `reason` ‚Äî `primerStatus` set to `error` |
| `TrackingDigestPublished` | kind-30841 signed, dual-written, and relayed | `event_urn`, `slug`, `d_tag` |
| `TrackingDigestPublishSkipped` | `nostr_publish` off or bridge secrets absent | `slug`, `reason` (I11) |
| `TrackingDigestPublishFailed` | Bridge `track` subcommand errors | `slug`, `reason` ‚Äî fail-open, scan unaffected |
| `ProjectUntracked` | Project no longer present under any `ScanRoot` | `project_urn`, `slug` ‚Äî metric series cleared |

All events are emitted through the ADR-005 observability middleware as plain domain projections in JSON. DDD-004 JSON-LD encoding is applied as an opt-in surface when `[linked_data]` is enabled; the `routes/projects.js` list and detail surfaces emit JSON-LD when the client sends `Accept: application/ld+json` and the linked-data gate is on.

## Anti-Corruption Layer

Two external systems sit at the edge of this domain, and each gets a thin translation layer so the domain expresses only its own language.

**Git ‚Üí domain**: `gitMetadata(repoPath)` (`lib/project-tracker.js`) is the ACL over git. It shells `git` via `execFile` and translates raw porcelain into the domain vocabulary: `{branch, lastCommitIso, lastCommitAgeSec, commits30d, commitDays, remote, language}`. The domain never sees git plumbing; it sees a `TrackedProject` projection. A git invocation that times out or errors is translated into a per-repo skip (I06), not a domain exception. The `language` field is a heuristic over the repository contents, not a git fact ‚Äî it is computed here and carried as a projection attribute.

**GitHub ‚Üí domain**: the `GithubEnrichmentPort` translates `gh` CLI JSON output into the two enrichment fields (`openIssues`, `stars`). It is the only place GitHub's schema is read. If the gate is off, the token is absent, or the remote is not a GitHub remote, the port returns a null overlay and emits `GithubEnrichmentSkipped` ‚Äî the `TrackedProject` projection is complete without enrichment, with `openIssues` and `stars` left `null`.

**Nostr ‚Üí domain**: this domain does not parse inbound nostr. It produces an unsigned `TrackingDigest` and hands it across the `NostrBridgePort`; everything downstream (signing, event-id computation, NIP-33 addressing, pod-mailbox dual-write, relay transport) is owned by `services/nostr-pod-bridge` and DDD-003. The kind number 30841 and the tag shape are the only nostr-wire facts this domain knows, and they live in the digest payload contract, not in domain logic.

## Domain Rules (cross-aggregate)

- **R01**: Project tracking is read-mostly observability, not a new adapter slot. All durable state routes through existing slots ‚Äî primers/synopses through the memory slot, scan activity and domain events through the events slot. No new backend is opened (I04, I12).
- **R02**: The slug is the only project label that crosses the box boundary. Every surface that federates ‚Äî metrics, nostr tags, digest content ‚Äî uses the slug; the absolute path stays internal (I02, I03).
- **R03**: Every durable identifier is minted through `management-api/lib/uris.js`. There are no ad-hoc URNs anywhere in this domain (I01).
- **R04**: Every external hop is gated and fail-open. GitHub enrichment, primer inference, and nostr publication each self-gate at runtime on their manifest flag and required secret, and each fails open ‚Äî an off or failed hop degrades the projection gracefully and never aborts a scan or a route (I06, I07, I11).
- **R05**: The feature self-gates as a whole. `routes/projects.js` returns `503 {error: 'project_tracking disabled'}` when `manifest.project_tracking.enabled !== true`; `observability/project-metrics.js` registers its series only when `[project_tracking].metrics` is on. A disabled feature emits nothing and exposes no surface.
- **R06**: A scan is the unit of evidence. No `TrackedProject` projection exists without a `ProjectScan` that produced it, and exactly one scan receipt is emitted per pass (I10). A primer or digest references the project URN, which references the scan that materialised it.

## Migration and Coexistence

**helm coexistence**: helm (the Fastify/React project dashboard) is the conceptual source of the tracking model ‚Äî status grid, 30-day commit activity, primers/synopses, GitHub plus local repo sync. agentbox adopts that model and rejects the stack: no Fastify dashboard, no React, no bespoke datastore. The tracking model is re-expressed on the three existing substrates (URN grammar, port-bound telemetry, custom-kind nostr). There is no runtime dependency on helm and no shared code; the lineage is conceptual.

**Session-summary sibling**: the kind-30841 `TrackingDigest` is a deliberate sibling of the kind-30840 session summary (`services/nostr-pod-bridge`). The `track` subcommand mirrors `publish_session_summary` ‚Äî same dual-write-then-publish path, same fail-open posture, same pod-mailbox durability. The two kinds share the relay's `allowed_kinds` list and the same owner identity.

**Existing observability**: the `agentbox_project_*` series register on the shared registry that already backs `/metrics`. No new port is bound and no new exporter is added; a monitoring stack already scraping `/metrics` picks up the project series with no configuration change.

**Phased rollout**: the feature is gated off by default (`[project_tracking].enabled = false`). Metrics, GitHub enrichment, primer-on-scan, and nostr publication are each independently gated, so an operator can enable scanning and telemetry without enabling any external hop. The JSON-LD surfaces remain gated by `[linked_data]` independently of the feature gate.

## Open Questions

1. **Scan-root drift and the `Untracked` transition**: `clearProject(slug)` removes a project's metric series when it disappears from all scan roots. Whether a withdrawn project should also publish a tombstone kind-30841 (NIP-33 deletion or an empty-content replacement under its `d`-tag) so the mesh reflects the withdrawal, or whether the digest should simply be left to age, is deferred to ADR-035 authorship. The current model clears local telemetry but leaves the last federated digest in place.

2. **Primer staleness policy**: a `ProjectPrimer` is generated from a project's git facts at a point in time; subsequent commits drift it. Whether `primer_on_scan` should regenerate on every scan, only on a commit-delta threshold, or only on explicit `force`, is a cost-versus-freshness trade-off deferred to ADR-035. The aggregate carries `scannedAt` and the primer carries its own write timestamp, which is sufficient to compute staleness once a policy is chosen.

3. **Multi-owner federation**: this domain assumes a single `AGENTBOX_PUBKEY` owner per container (I08). In a future multi-profile container where several sovereign identities each watch overlapping scan roots, the `owner_did` scoping is already correct (each `TrackedProject` URN is scoped to its owner pubkey), but the metric `project` label would collide across owners for the same slug. A per-owner label or a slug-plus-owner composite is reserved for that scenario and is out of scope here.

4. **GitHub enrichment beyond issues and stars**: the enrichment overlay is intentionally minimal (open issues, stars) to keep the single external hop cheap and rate-limit-friendly. Richer signals (PR counts, release cadence, contributor counts) are deliberately out of v1 scope; adding them is a port-contract extension, not a domain change.
